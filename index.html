  • a List
    • function prepend(entry, list){
return {value: entry, rest: list};
}

function arrayToList(array){
  var list = null;
  for(var i = array.length - 1; i >= 0; i--)
    list = prepend(array[i], list);
  return list;
}

function listToArray(list){
  var array = [];
  for(var node = list; node; node = node.rest)
    array.push(node.value);
  return array;
}

function nth(list, pos){
  if(!list)
    return undefined;
  else if(pos == 0)
    return list.value;
  else
    return nth(list.rest, pos-1);
}
  • Deep comparison
    • function deepEqual(a, b){
  if (a === b)
    return true;
  if(typeof a != "object" || typeof b != "object" || a == null || b == null)
    return false;

  var propA = 0, propB = 0;
  for (var prop in a)
    propA++;
  for(var prop in b){
    propB++;
    if(!(prop in a) || !deepEqual(a[prop], b[prop]))
      return false;
  }

  return propA == propB;
}

Day 9: Note on vertical alignment in a div

Aug 28

There are a few different solutions I tried when implementing the welcome screen and getting the text to center vertically and horizontally:

  • Absolute positioning with a combination of top, bottom, left and right attributes set to zero
    • Wasn't the best solution since I needed the div to be aware of the elements around it
  • Setting the line-height
    • Only good when the div size is known before hand, didn't fit with the goal of the page trying to be as fluid as possible
  • Using relative positioning and transforms
    • Not effective when the div takes up the screen
  • Div with the display set to table-cell and using vertical align
    • As mentioned previously, the vertical-align attribute was not intended to vertically align text in block elements, but does so when the block element is a table. This was the best working solution for the welcome screen at this time

I'm interested to see what the finalized versions of Flexbox and Grid Layout look like and see if they can provide a better solution to the one in place

Day 8: Minor Changes and a Reflection

Aug 27

It's been one week since I started this project. I'm a big believer in reflection to help reinforce lessons that have been learned. Comparing the first entry and layout with the current set leaves me with a satisfied feeling.

Some of the most important things I've learned include:

  • Knowing the difference between Block and Inline elements and their intended purpose
  • Paddings and Margins: these are some of the most important CSS attritubes in terms of positioning elements just the way you want
  • Layouts: starting out with a fixed layout is fine, but isn't as flexible as a liquid layout. The later makes it easier to handle various browser window sizes, but make sure to set the min-width and max-width accordingly to ensure readability
  • Viewports: relatively new measurement unit where 1vh = 1% of the browser height and 1vw = 1% of the browser width. Used this to create the welcome screen (provides the flexibility to adjust based on the browser window size)

I think the next hurdle to climb would be getting a better grasp of JavaScript. I'll continue to document my progress on this page and see what kind of new features can be incorportated via Javascript.
